					<div id="mw-content-text" class="mw-body-content"><div class="mw-content-ltr mw-parser-output" lang="pl" dir="ltr"><figure typeof="mw:File/Thumb"><a href="/wiki/Plik:URI_Venn_Diagram.svg" class="mw-file-description"><img src="//upload.wikimedia.org/wikipedia/commons/thumb/f/ff/URI_Venn_Diagram.svg/200px-URI_Venn_Diagram.svg.png" decoding="async" width="200" height="121" class="mw-file-element" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/f/ff/URI_Venn_Diagram.svg/300px-URI_Venn_Diagram.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/f/ff/URI_Venn_Diagram.svg/400px-URI_Venn_Diagram.svg.png 2x" data-file-width="800" data-file-height="482" /></a><figcaption>Relacje między <a href="/wiki/Uniform_Resource_Identifier" title="Uniform Resource Identifier">URI</a> a URL i <a href="/wiki/Uniform_Resource_Name" title="Uniform Resource Name">URN</a></figcaption></figure>
<p><b>URL</b> (<a href="/wiki/J%C4%99zyk_angielski" title="Język angielski">ang.</a> <i>uniform resource locator</i>) – ujednolicony format adresowania (określania lokalizacji) zasobów (informacji, danych, usług) stosowany w <a href="/wiki/Internet" title="Internet">Internecie</a> i w <a href="/wiki/Lokalna_sie%C4%87_komputerowa" title="Lokalna sieć komputerowa">sieciach lokalnych</a>. Standard URL opisany jest w dokumencie<span class="harvard-citation"><a href="#CITEREFRFC_1738">RFC 1738 ↓</a></span>.
</p><p>Tak zwany adres URL najczęściej kojarzony jest z adresami stron <a href="/wiki/World_Wide_Web" title="World Wide Web">WWW</a>, ale ten format adresowania służy do określania lokalizacji wszelkich zasobów dostępnych w Internecie.
</p>
<meta property="mw:PageProp/toc" />
<div class="mw-heading mw-heading2"><h2 id="Opis_formatu">Opis formatu</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Uniform_Resource_Locator&amp;veaction=edit&amp;section=1" title="Edytuj sekcję: Opis formatu" class="mw-editsection-visualeditor"><span>edytuj</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Uniform_Resource_Locator&amp;action=edit&amp;section=1" title="Edytuj kod źródłowy sekcji: Opis formatu"><span>edytuj kod</span></a><span class="mw-editsection-bracket">]</span></span></div>
<div class="mw-heading mw-heading3"><h3 id="Elementy_adresu">Elementy adresu</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Uniform_Resource_Locator&amp;veaction=edit&amp;section=2" title="Edytuj sekcję: Elementy adresu" class="mw-editsection-visualeditor"><span>edytuj</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Uniform_Resource_Locator&amp;action=edit&amp;section=2" title="Edytuj kod źródłowy sekcji: Elementy adresu"><span>edytuj kod</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>URL składa się ze schematu (ang. <i>scheme</i>), czyli części określającej rodzaj zasobu/usługi, <a href="/wiki/Dwukropek" title="Dwukropek">dwukropka</a> i części zależnej od rodzaju zasobu (ang. <i>scheme-specific part</i>) zawsze zaczynającej się od ukośnika <b>/</b>, czyli ma postać:
</p>
<pre>&lt;schemat&gt;&#160;: &lt;specyficzna dla schematu część hierarchiczna&gt;
</pre>
<ul><li>Przykład prostego adresu URL:</li></ul>
<pre>http://www.wikipedia.org/wiki/URL
\__/   \_______________/\_______/
 |            |             |
protokół     <a href="/wiki/Host" title="Host">host</a>        ścieżka
        (adres serwera)  do zasobu
</pre>
<ul><li>Przykład rozbudowanego adresu URL:</li></ul>
<pre>http://hans:geheim@www.example.org:8080/demo/example.cgi?land=de&amp;stadt=aa
\__/   \__/ \____/ \_____________/ \__/\_______________/ \______________/
 |      |     |           |         |          |                 |
 |    login   |          host      port     ścieżka           ścieżka
protokół    hasło  (adres serwera)         do zasobu        wyszukiwania
</pre>
<p>Często <a href="/wiki/Oprogramowanie" title="Oprogramowanie">oprogramowanie</a>, szczególnie <a href="/wiki/Przegl%C4%85darka_internetowa" title="Przeglądarka internetowa">przeglądarki internetowe</a>, akceptuje także niepoprawne formy adresów – pominięty separator <code>//</code> czy określenie protokołu <code>http://</code>, np.:
</p>
<pre>wikipedia.org/wiki/URL
</pre>
<div class="mw-heading mw-heading3"><h3 id="Rodzaj_zasobu_(schemat)"><span id="Rodzaj_zasobu_.28schemat.29"></span>Rodzaj zasobu (schemat)</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Uniform_Resource_Locator&amp;veaction=edit&amp;section=3" title="Edytuj sekcję: Rodzaj zasobu (schemat)" class="mw-editsection-visualeditor"><span>edytuj</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Uniform_Resource_Locator&amp;action=edit&amp;section=3" title="Edytuj kod źródłowy sekcji: Rodzaj zasobu (schemat)"><span>edytuj kod</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Nazwa schematu (usługi) może składać się z małych liter, cyfr, plusa, dywiza oraz kropki. Ze względu na możliwość występowania pomyłek, na ogół akceptowane są przez <a href="/wiki/Oprogramowanie" title="Oprogramowanie">oprogramowanie</a> także <a href="/wiki/Majusku%C5%82a" title="Majuskuła">wielkie litery</a>. Popularne rodzaje zasobów:
</p>
<ul><li><a href="/wiki/Protok%C3%B3%C5%82_transferu_plik%C3%B3w" title="Protokół transferu plików">ftp</a> – usługa FTP</li>
<li><a href="/wiki/HTTP" title="HTTP">http</a> – usługa WWW</li>
<li><a href="/wiki/HTTPS" title="HTTPS">https</a> – usługa szyfrowanego protokołu HTTP</li>
<li><a href="/wiki/Telnet" title="Telnet">telnet</a> – usługa telnet</li>
<li><a href="/wiki/Network_News_Transfer_Protocol" title="Network News Transfer Protocol">nntp</a> – usługa Usenet</li>
<li><a href="/wiki/Wide_Area_Information_Server" title="Wide Area Information Server">wais</a> – usługa WAIS</li>
<li><a href="/wiki/Gopher" title="Gopher">gopher</a> – usługa Gopher</li>
<li><a href="/wiki/Usenet" title="Usenet">news</a> – usługa Usenet</li>
<li><a href="/wiki/Poczta_elektroniczna" title="Poczta elektroniczna">mailto</a> – poczta elektroniczna</li>
<li><a href="/w/index.php?title=File_(protok%C3%B3%C5%82)&amp;action=edit&amp;redlink=1" class="new" title="File (protokół) (strona nie istnieje)">file</a> – <a href="/wiki/Plik_danych" title="Plik danych">plik</a> lokalny</li>
<li><a href="/wiki/Extensible_Messaging_and_Presence_Protocol" title="Extensible Messaging and Presence Protocol">jid</a> – Jabber/XMPP</li></ul>
<div class="mw-heading mw-heading3"><h3 id="Część_zależna_od_rodzaju_zasobu"><span id="Cz.C4.99.C5.9B.C4.87_zale.C5.BCna_od_rodzaju_zasobu"></span>Część zależna od rodzaju zasobu</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Uniform_Resource_Locator&amp;veaction=edit&amp;section=4" title="Edytuj sekcję: Część zależna od rodzaju zasobu" class="mw-editsection-visualeditor"><span>edytuj</span></a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Uniform_Resource_Locator&amp;action=edit&amp;section=4" title="Edytuj kod źródłowy sekcji: Część zależna od rodzaju zasobu"><span>edytuj kod</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Część zależna od rodzaju usługi zwykle przybiera jedną z postaci:
</p>
<ul><li>W przypadku zasobów będących plikami:</li></ul>
<pre>//adres_serwera:port/sciezka_dostępu
</pre>
<dl><dd>Jeżeli <a href="/wiki/Port_protoko%C5%82u" title="Port protokołu">port</a> jest standardowy dla danego rodzaju zasobu, jest pomijany i stosuje się formę uproszczoną:</dd></dl>
<pre>//adres_serwera/sciezka_dostępu
</pre>
<dl><dd>W niektórych przypadkach (np. usługa FTP) może być wymagane podanie nazwy użytkownika i hasła:</dd></dl>
<pre>//nazwa_użytkownika:hasło@adres_serwera/sciezka_dostępu
</pre>
<dl><dd>ale najczęściej zarówno <code>nazwa_użytkownika</code>, jak i <code>hasło</code> nie są wymagane i mogą być pominięte. Należy także zauważyć, że podawanie hasła w ten sposób może doprowadzić do naruszenia bezpieczeństwa systemu – w łatwy sposób niepowołana osoba może takie hasło przejąć.</dd></dl>
<ul><li>W przypadku zasobów niebędących plikami (<a href="/wiki/Konto_shell" title="Konto shell">konta shellowe</a>, <a href="/wiki/Adres_poczty_elektronicznej" title="Adres poczty elektronicznej">adresy email</a> itp.):</li></ul>
<pre>nazwa_uzytkownika@adres_serwera
</pre>
